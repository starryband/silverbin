# Contributing
<sup>I'm new to this type of stuff so don't hate on it</sup>
## Getting Started
1. Fork the repository
2. Clone your fork
3. Create a new branch
 - feature/feature-name
 - fix/bug-description
## Code Guidelines
- Maintain consistent formatting
- Keep functions small and readable
- Avoid using unnecessary dependencies
## Submitting Changes
1. Commit your changes with *clear* messages
2. Push to your fork
3. Open a PR
## Pull Requests
- Describe what your PR does
- Link any related issues
- Avoid mixing unrelated changes with your PR
## Issues
- Only use these to report bugs or suggest features
- Be detailed (use screenshots, logs, etc)
## Notes
This engine is still in early development, so bugs are to be expected.
